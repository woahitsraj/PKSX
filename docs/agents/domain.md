# Domain Docs

Before planning or implementing, read:

- `CONTEXT.md` at the repo root, if present
- Relevant ADRs under `docs/adr/`, if present

This repo uses a single domain context.

Use terms from `CONTEXT.md` in issue titles, implementation notes, tests, and PR summaries. If a needed domain term is missing, resolve it through `/grill-with-docs` instead of inventing synonyms.
